# /check-schema — Supabaseスキーマとapp.jsの整合性チェック

`supabase_schema.sql` と `app.js` の間でテーブル名・カラム名・操作が一致しているかを確認する。

## チェック手順

1. `supabase_schema.sql` を読み、テーブル名・カラム名・RLSポリシーを把握する
2. `app.js` を読み、Supabaseへのクエリ（`.from()` `.select()` `.insert()` `.update()` `.delete()`）をすべて抽出する
3. 以下の不整合を報告する：
   - app.js が参照しているテーブル名がスキーマに存在しないもの
   - app.js が参照しているカラム名がスキーマに存在しないもの
   - スキーマのRLSポリシーで禁止されている操作をapp.jsが行っているもの
   - スキーマにはあるがapp.jsで使われていないテーブル・カラム（未使用の可能性）
4. 問題があれば修正案を提示する。問題がなければ「整合性OK」と報告する
