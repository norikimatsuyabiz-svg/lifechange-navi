# spec.md — ライフチェンジ手続きナビ 仕様書

## 1. 目的・利用シーン

### 目的
結婚・転職など生活の節目に発生する名義変更・住所変更などの手続きを、依存関係（「Aが終わるまでBに着手できない」）とともに整理し、抜け漏れなく完了まで導くサポートツール。

### 利用シーン
- 入籍直後に「何から手をつければいいか」を確認するとき
- あるタスクが完了したあと「次に何が解放されるか」を確認するとき
- 全体の進捗を把握したいとき

---

## 2. データ項目

### ヒアリング回答（answers）

| 項目 | キー | 値 | 条件 |
|---|---|---|---|
| 改姓有無 | `kasei` | `yes` / `no` | 常に表示 |
| 引っ越し有無 | `hikkoshi` | `yes` / `no` | 常に表示 |
| 自動車保有 | `kuruma` | `yes` / `no` | 常に表示 |
| パスポート保有 | `passport` | `yes` / `no` | `kasei === 'yes'` のとき |
| 資格保有 | `shikaku` | `string[]` 例: `['ishi','kango']` | `kasei === 'yes'` のとき |
| 扶養追加 | `fuyou` | `yes` / `no` | 常に表示 |
| 口座・カード数 | `banks` | `few` / `some` / `many` | 常に表示 |

### タスク1件

| 項目 | キー | 必須 | 備考 |
|---|---|---|---|
| ID | `id` | ○ | 例: `kon`, `bank_main` |
| タスク名 | `name` | ○ | 例: 婚姻届を提出する |
| カテゴリ | `cat` | ○ | 役所／銀行・金融／保険／免許／住居／会社／カード・通信／資格 |
| 担当 | `who` | ○ | 本人／妻／二人 |
| 依存リスト | `deps` | ○ | 先に完了すべきタスクのID配列（空配列も可） |
| 優先度 | `priority` | ○ | 1=高／2=中／3=低 |
| 補足メモ | `note` | | 任意 |
| 期限 | `deadline` | | 任意・ISO文字列 例: `2026-07-31` |
| 窓口メモ | `memo` | | 任意・持ち物・電話番号など |
| 公式URL | `url` | | 任意・手続きの公式サイト |

## 3. 画面構成

### 画面遷移

```
screen-welcome（ウェルカム）
  └─ イベント選択
       └─ screen-chat（ヒアリング）
            └─ 全問回答後、自動遷移
                 └─ screen-project（プロジェクト）
```

### screen-welcome（ウェルカム画面）
- アプリ名・説明文
- イベント選択ボタン（結婚・入籍 / 転職・退職 / 引っ越しのみ）
- 未実装イベントには「近日追加」バッジを表示

### screen-chat（ヒアリング画面）
- チャット形式で1問ずつ表示
- 回答済みの質問は吹き出し履歴として残る
- 単一選択（`type: 'single'`）と複数選択（`type: 'multi'`）に対応
- 条件付き質問（`cond`）あり：改姓時のみパスポート・資格を表示
- 複数選択は「決定する →」ボタンで確定（選択なしでスキップ可）
- 進捗表示（例: 3 / 6）
- 全問回答後、自動的にプロジェクト画面へ遷移

### screen-project（プロジェクト画面）
- **ヘッダー**：プロジェクト名・全体進捗バー
- **メトリクス（4枚）**：進捗% / 完了件数 / 着手可能件数 / ブロック中件数
- **フィルター**：カテゴリ・担当者で絞り込み
- **タスクリスト（3セクション）**

| セクション | 表示条件 |
|---|---|
| 今すぐ着手できる | 未完了 かつ 依存タスクがすべて完了済み |
| ブロック中 | 未完了 かつ 依存タスクが1件以上未完了 |
| 完了 | 完了済み |

- **タスクカード**：チェックボタン / タスク名 / ブロック理由 / 担当バッジ / カテゴリバッジ / 編集ボタン（鉛筆アイコン）

## 4. CRUD操作フロー

| 操作 | トリガー | 処理 | 結果 |
|---|---|---|---|
| プロジェクト生成 | ヒアリング全問回答 | answersからタスク一覧と依存関係を生成 | プロジェクト画面へ遷移 |
| タスク完了 | チェックボタン押下 | doneSetにidを追加・保存 | 依存先タスクが解放される |
| 完了解除 | 完了済みチェック押下 | 確認ダイアログ→依存タスクも連動解除 | 依存タスクが再びブロックされる |
| タスク編集 | 鉛筆アイコン押下 | モーダルを開く | deadline / memo / url を入力・保存 |
| フィルター | フィルターボタン押下 | activeFilterを変更・再描画 | 絞り込み表示 |
| リセット | 「最初から」ボタン押下 | localStorage削除・状態初期化 | ウェルカム画面へ戻る |

## 5. 永続化方針

### localStorageキー

| キー | 型 | 内容 |
|---|---|---|
| `lifechange_v1` | JSON文字列 | event / answers / done / tasks をまとめて保存 |

### 保存形式

```json
{
  "event": "marriage",
  "answers": { "kasei": "yes", "hikkoshi": "yes", "shikaku": ["ishi"] },
  "done": ["kon", "juminhyo"],
  "tasks": [ { "id": "kon", "name": "婚姻届を提出する", "deps": [] } ]
}
```

### 保存タイミング

| タイミング | 処理 |
|---|---|
| ヒアリング中 | 各回答ごとに保存（途中離脱に備える） |
| プロジェクト生成時 | タスク一覧確定後に保存 |
| タスク完了・解除時 | 操作ごとに即座に保存 |
| タスク編集時 | モーダルの「保存」ボタン押下時に保存 |

### 復帰動作
- ページロード時に `loadStorage()` を実行
- `tasks` が存在する場合はプロジェクト画面を直接表示
- `tasks` が空の場合はウェルカム画面を表示

---

## 6. 初期データ

サンプルデータは不要。ヒアリングに答えることでタスクが自動生成されるため。

- localStorage が空の場合 → ウェルカム画面を表示
- tasks が保存済みの場合 → プロジェクト画面を直接表示（途中離脱からの復帰）

---

## 7. 対象外の機能

| 機能 | 理由 |
|---|---|
| カレンダー表示 | 規模が大きくなるため将来拡張候補 |
| 転職・退職イベント | 将来拡張として設計済み・現版は未実装 |
| 引っ越しのみイベント | 将来拡張として設計済み・現版は未実装 |
| タスクの手動追加・削除 | ヒアリング生成で対応 |
| クラウド保存・他端末共有 | サーバーが必要なため対象外 |
| OCR（書類の自動読み取り） | 外部APIが必要なため対象外 |
| モバイル対応 | PCブラウザでの利用を前提 |
