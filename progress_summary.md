# ライフチェンジ手続きナビ — 開発まとめ

## 作ったもの

**結婚・入籍に伴う行政手続きを、依存関係つきで整理・管理するWebアプリ**

URL: Vercel にデプロイ済み  
リポジトリ: GitHub (norikimatsuyabiz-svg/lifechange-navi)

---

## 問題意識

結婚・入籍時の手続き（婚姻届・改姓・引っ越し・銀行口座変更など）は件数が多く、「何から始めればいいか」「どの手続きが先に必要か」が分かりづらい。チェックリストを自分で作るのが面倒で、手続き漏れや順番ミスが起きやすい。

## 実装した機能

### コア機能
- **ヒアリングチャット**：改姓・引っ越し・自動車・資格など6〜8問に答えるだけで手続きタスクを自動生成
- **依存関係管理**：「婚姻届が完了しないと改姓手続きができない」など、deps（依存）を持つタスクを自動ブロック・アンロック
- **完了チェック**：完了時に依存タスクを連鎖解除、取り消し時は依存タスクも巻き戻す
- **カテゴリ・担当フィルター**：役所・銀行・免許など、カテゴリや担当者で絞り込み
- **タスク編集モーダル**：期限・窓口メモ・公式URLを各タスクに記録
- **Supabase永続化**：匿名ユーザーとして自動ログイン、データは projects / project_tasks / project_done の3テーブルで管理

### カレンダー機能
- **リスト/カレンダー切替**：プロジェクト画面でビューを切り替え
- **期限の可視化**：期限設定済みタスクがカレンダー上にチップ表示
- **日付クリックで期限設定**：未設定タスクをその日に割り当て
- **ドラッグ&ドロップ**：タスクチップをつかんで別の日付にドロップして期限移動
- **期限解除・タスク削除**：日付モーダルから操作可能
- **リアルタイム反映**：操作後にリロードなしで即時更新

### AI活用機能（Vercel Functions経由でAPIキーを保護）
- **✨ タスク個別チャット**：各タスクカードの✨ボタンで、そのタスクに特化したAIと会話
- **📋 メモに保存**：AI返答をそのままタスクのメモ欄に追記
- **✂️ 要約して保存**：AI返答を3行以内の箇条書きに要約してからメモに保存
- **✨ AIアドバイス**：全タスク・完了状況・期限を渡し「今週の優先タスク」「注意点」を構造化して提案

### UX
- **初回チュートリアル**：初回起動時にのみ5ステップの使い方ガイドを表示（LocalStorageで管理）
- **「使い方」ボタン**：ヘッダーからいつでもチュートリアルを再表示
- **ダークモード対応**：`prefers-color-scheme: dark` で自動切替
- **資格選択の「なし」オプション**：複数選択の資格質問に「なし（不要）」を追加し、排他制御を実装

## 工夫した点

### セキュリティ設計
- Anthropic APIキーはフロントエンドに一切含めず、**Vercel Functions (`api/chat.js`) でプロキシ**
- Supabaseは`publishable key`のみ使用、`service_role`キーはフロントに出さない
- RLS（Row Level Security）でユーザーは自分のデータのみアクセス可能

### Claudeとの試行錯誤
- チャット返答を`onclick`属性に`JSON.stringify`で埋め込んでいたため、返答に引用符や改行が含まれるとボタンが動かないバグを発見 → `addEventListener`＋クロージャに変更
- カレンダービューで期限変更後にリロードしないと反映されない問題を発見 → `renderProject()`と`renderCalendar()`を`renderView()`で統合
- ドラッグ&ドロップは`draggable="true"`＋`ondragstart/ondragover/ondrop`のVanilla JS実装で対応（ライブラリ不使用）

### Claudeの活用方法の工夫
- **3つのAPIモードを1エンドポイントで管理**：`mode: 'chat'`（タスク個別）、`mode: 'advisor'`（進捗分析）、`mode: 'summarize'`（要約）をリクエストボディで切り替え
- **コンテキストを渡してAIの精度を上げる**：タスク名・カテゴリ・完了状況・期限・ブロック状態をすべてアドバイザーに渡し、的確な優先順位提案を実現
- **要約保存で実用性を高める**：長いAI返答を3行の箇条書きに自動要約し、メモとして保存できる機能でAIの回答を実務に直結させた

## input / output 例

### ヒアリング（input）
```
改姓: あり
引っ越し: あり
自動車: なし
パスポート: あり
資格: 看護師免許
扶養: なし（共働き）
口座・カード: 3〜5つ
```

### 生成タスク一覧（output）
```
✅ 婚姻届を提出する（役所・二人）
⬜ 転入届を提出する（役所・二人）← 婚姻届が完了したらアンロック
⬜ 住民票の写しを取得する（役所・妻）
⬜ 運転免許証の氏名・住所変更（免許・妻）
⬜ マイナンバーカードの氏名変更（役所・妻）
⬜ メイン銀行口座の氏名変更（銀行・妻）
⬜ パスポートの氏名変更申請（免許・妻）
⬜ 看護師免許の氏名変更を申請する（資格・妻）
⬜ 年末調整で配偶者控除を申告する（会社・本人）
… 計18件
```

### AIアドバイス（output例）
```
## 今すぐ着手すべき優先タスク
- 婚姻届を提出する（すべての手続きの起点）

## 今週中に済ませたいこと
- 住民票の取得（氏名変更手続きで繰り返し必要）

## 注意点
- パスポートの変更には運転免許証の変更が先に必要
- 看護師免許の変更には戸籍謄本が必要なため早めに準備を
```
